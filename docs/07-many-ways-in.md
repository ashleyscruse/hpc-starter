# More Than One Way In

Everything you did today, you typed into a terminal. That's not an accident. **A supercomputer is, at its core, a terminal-based system.** The commands are the real controls.

But the terminal isn't the only door.

| Door | What it feels like | What's really happening |
|---|---|---|
| **Terminal (SSH)** | Type commands directly | You're talking straight to the machine |
| **Tapis / web portal** | Click around in a browser | Buttons run commands for you |
| **Jupyter notebooks** | Cells, output, plots | A friendly window onto the same system |

Here's the key idea: when you use **Tapis or a Jupyter notebook**, you are on the **exact same supercomputer** you used today. The graphical interface is just a friendlier window. Under the hood, it's still the terminal-based machine, running the same kinds of commands you just learned.

So why learn the terminal first?

- The GUI is **convenient**. The terminal is the **truth underneath**.
- When something doesn't work in the nice interface, knowing the terminal lets you understand and fix it.
- You'll do your ML work in **Jupyter** later, and now you'll know what's actually happening when you click "run."

You didn't just learn some commands. You learned what the supercomputer really is, and every friendlier tool is just a door into it.
